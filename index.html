Overview

A software is developed which would help in the analysis of large amounts of unindexed data written in English language. The software can be accessed by clicking here. The input text is first split into several small sections. This splitting happens automatically if there are several topic markers ("T o p i c #") or sub-topic markers ("     #.") present (# is any number). In the absence of sub-topic markers splitting takes place according to newline characters. Users can specify the number of paragraphs that they want to be clubbed in one section. After this section splitting, specified number of keywords are extracted from each section. A summary comprising the specified number of sentences is also extracted from each section. Finally, each section is compared with all the other sections to calculate the degree of similarity among them and the specified number of most similar sections are displayed for every section.

The above pipeline is developed by three widely used Python libraries - Gensim, NLTK and spaCy. The pre-processing with NLTK and spaCy includes tokenization, stopwords removal, multiple whitespaces removal, accented characters removal, Parts of Speech (POS) tagging and lemmatization. Pre-processing with Gensim also performs these tasks except accented character removal and POS tagging. Keywords extraction and summarization with Gensim is done by using its implementation of TextRank. In case of NLTK and spaCy there are two bag-of-words based models to perform these tasks - a word occurence frequency based model and a tf-idf based model. Text similarity with Gensim is calculated by using Latent Semantic Indexing. With NLTK and spaCy this is calculated by a cosine based approach.

NLTK and spaCy perform the same pre-processing tasks and spaCy takes much less time as compared to NLTK. The pre-processing tasks performed by Gensim are different from those of NLTK and spaCy. There is no POS tagging happening with Gensim. So the lower time with Gensim does not mean that its pre-processing is more efficient than that of NLTK and spaCy.

The tf-idf based keywords extraction and summarization with spaCy takes lesser time as compared to TextRank with Gensim.
